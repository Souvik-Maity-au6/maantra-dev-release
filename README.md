# Maantra – Public Release Branch

## About Maantra

Maantra is a spiritual service booking platform designed to help users easily book trusted pandits for puja and religious services. The app connects users, pandits, and admins in one organized flow so that puja bookings, assignments, payments, service updates, and support can be managed smoothly.

The goal of Maantra is to make the traditional puja booking process more accessible, transparent, and convenient through a modern mobile-first experience.

## Purpose of This Release

This release branch is created for the first public/internal MVP testing phase of the Maantra mobile application.

The purpose of this release is to test the core user and pandit workflows before moving towards a wider production release. This branch includes the basic structure, main app flows, API integrations, payment flow, booking management, and notification setup required for MVP validation.

## What This Release Includes

* User login with mobile OTP
* User service browsing
* Puja booking flow
* Online payment flow
* Payment pending and payment success handling
* User booking details and booking history
* Booking cancellation request and refund review flow
* Pandit login and verification flow
* Pandit dashboard
* Pandit booking management
* Pandit wallet and withdrawal request flow
* Support ticket flow
* Reviews and ratings
* Booking chat
* Push and in-app notification support
* Basic profile and address management
* App-wide UI/UX fixes for MVP testing

## Testing Focus

This release should be tested mainly for:

* Login and OTP flow
* User booking journey
* Payment and booking status updates
* Pandit assignment and acceptance flow
* Booking cancellation and refund status display
* Pandit wallet and withdrawal request
* Support ticket creation and reply flow
* Notification receive and tap navigation
* General app stability and UI issues

## Release Note

This is an MVP testing release. Some operations, such as refund processing and pandit assignment, may still depend on admin-side manual actions.

The main purpose of this branch is to validate the complete booking lifecycle and collect feedback before the next stable production release.
